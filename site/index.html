One request, from input to audit evidence

1. ValidatePydantic schema rejects unknown or invalid fields.
2. Build featuresVersioned feature contract creates the model row.
3. ScoreThe champion model returns a probability.
4. Apply policyA separate policy maps score and gates to an action.
5. Route reviewBorderline or safety-sensitive cases go to a human.
6. RecordDecision, audit, model, policy, and schema identifiers are emitted.

Reproduce the complete release

git clone https://github.com/xm2325/regulated_ml_platform.git
cd regulated_ml_platform
python -m venv .venv
source .venv/bin/activate
pip install -r requirements-dev.txt
make all

make all verifies and restores the complete source snapshot when required, generates 5,000 synthetic customers, builds features, trains candidate models, selects the champion and threshold on validation data, evaluates the test set, creates governance and operational reports, builds this page, and runs the tests.

Read the evidence in this order

  1. Release approval pack: overall decision and control table.
  2. Promotion gate: exact pass/review criteria.
  3. Model evaluation: independent test metrics and confidence intervals.
  4. Calibration and segment behaviour: probability and group diagnostics.
  5. Model contract: versioned input, output, policy, and prohibited-use rules.
  6. Machine-readable evidence: metrics used to build this page.
